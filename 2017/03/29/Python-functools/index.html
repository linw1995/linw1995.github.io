lru_cache

Least Recently Used(LRU)

装饰器，用来缓存函数对应不同参数的返回结果。

可以提供maxsize=None与typed=False这两个参数，
参数maxsize为最多缓存的次数，如果为None，则无限制，设置为2n时，性能最佳，特别适合于耗时的函数。
typed=True，则不同参数类型的调用将分别缓存，例如 f(3) 和 f(3.0)。

被装饰的函数会有cache_clear和cache_info这两个方法，分别用于清除缓存和查看缓存信息

py2与py3的区别

首先先看下functools在py2与py3间的区别：

函数 py2 py3 描述
cmp_to_key ✔ ✔ 把老式比较函数转换为新式比较方式（包装 -> 比较 -> 解包）
total_ordering ✔ ✔ 利用类定义中仅有的一或多个比较函数，来补全剩余的比较函数
reduce ✔ ✔ 与reduce函数一致
partial ✔ ✔ 用预先准备的参数包装该函数，以后每次调用只需要补充剩余的参数即可
partialmethod ✘ ✔ 同上，但只适用于类方法
update_wrapper ✔ ✔ 更新包装后的函数，即函数名、文档、等等，有利于调试
wraps ✔ ✔ 以上函数的装饰器版本，更方便使用
lru_cache ✘ ✔ 装饰器，缓存函数的调用结果，有限地（可无限）保存最近的调用结果
singledispatch ✘ ✔ 把函数转换成单分派泛函数

py3多了lru_cache, partialmethod, singledispatch这三个函数
